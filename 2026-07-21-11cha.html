📚 도서 반납일 계산 — WEEKDAY + IF (오늘 최고 강조)

무엇을 감싸나 요일은 반납일로 따집니다. 반납일 = 대출일 + 대여기간. 이 반납일을 WEEKDAY(대출일+기간, 2)로 감싸요. 대출일로 요일을 구하는 게 가장 많은 실수예요.
요일 번호 WEEKDAY 옵션 2번이면 월요일=1 … 토요일=6, 일요일=7.
구간 3개 · IF 2개 ①토요일(6)이면 주말을 건너뛰어 월요일 → 기간에 +2일 ②일요일(7)이면 → +1일 ③평일이면 그냥 대여기간만. =IF(WEEKDAY(대출+기간,2)=6, 대출+기간+2, IF(WEEKDAY(대출+기간,2)=7, 대출+기간+1, 대출+기간))
패턴은 동일 도서·장난감·장비 대여 전부 같은 패턴. 대여기간(10일·5일·3일)만 바뀝니다. 채우기 핸들로 나머지 채우기.
시험 예시: "대출일 이용해 반납일 계산, 토·일이면 다음 월요일" — 정말 많이 나오고 다들 여기서 막힙니다. 안 되면 넘어갔다 오세요.

⏱ 시간에 분 더하기 — TIME 함수

왜 TIME 5분은 시간값으로 0.00347…이라 손으로 못 씁니다. TIME(시,분,초)가 이 값을 만들어줘요. 5분이면 TIME(,5,)(시·초는 비움), 1시간이면 TIME(1,,).
주행기록 + 5분 "마스터로 끝나는 선수는 기록에 5분 추가" → 기록 = 도착-출발(시간 데이터는 그냥 빼기). 여기에 +TIME(,5,). =IF(RIGHT(소속,3)="마스터", (도착-출발)+TIME(,5,), 도착-출발)
함수 추가 금지 시험이 "TIME 이용"이라 하면 반드시 TIME을 쓰고, 지정 함수 외에는 추가하지 않습니다.
시험 예시: TIME이 "시간 형식으로 되돌리는" 원래 역할과 다르게 "일정 시간을 더하는" 용도로 나와 어렵게 느껴집니다.

🏆 조건 최고값의 항목 — DMAX → MATCH → INDEX

1단계 DMAX "재무팀 중 상여금이 가장 높은 값"을 먼저 구합니다. DMAX(전체표, "상여금", 조건목록). DB함수라 머리글 포함 전체 테이블과 조건 목록(소속팀=재무팀)을 씁니다.
2단계 MATCH 그 최고값이 몇 번째 행인지 찾습니다. MATCH(DMAX값, 상여금 레코드 범위, 0) — 검색범위는 머리글 빼고 레코드만(포함하면 행이 밀려 엉뚱한 값).
3단계 INDEX 그 행의 직위를 꺼냅니다. INDEX(직위 범위, 행번호) — 결과 열만 범위로 주면 열 번호는 생략.
시험 예시: "분류가 패션인 것 중 증감률 최고 항목" 등과 같은 패턴 — DMAX/MATCH/INDEX 세트로 반복됩니다.

🔤 HLOOKUP + RIGHT + 문자 vs 숫자(×1)

구조 상품코드 오른쪽 숫자(RIGHT(코드,1))로 기준표에서 포장방법을 찾습니다. HLOOKUP(RIGHT(코드,1), 기준표, 2, FALSE) — 첫 행 매칭, 머리글 제외, F4 절대, 정확히 일치.
빈 값이 나오면 RIGHT·LEFT·MID 결과는 문자. 기준표의 코드가 숫자면 안 맞아 결과가 안 나옵니다. 원본은 못 바꾸니 수식 쪽을 RIGHT(코드,1)*1로 숫자화(×1은 연산하면서 따옴표를 없앰).
점검 수식이 맞는데 값이 안 나오면 대화상자(fx)를 열어 실제로 들어간 값을 봅니다.
시험 예시: 코드 매칭 문제에서 문자/숫자 형 불일치는 단골 함정. ="0" 또는 ×1 둘 다 알아둡니다.

🏅 RANK + CHOOSE + IF/IFERROR (등급 표시)

순위 총결제액으로 RANK.EQ(값, 전체 F4, 0) — 0=내림차순(큰 값이 1등).
등급 CHOOSE(등수, "VIP","VIP","골드","골드","실버","실버") — 1·2등 VIP, 3·4 골드, 5·6 실버. 대문자·따옴표는 fx 대화상자로 편하게.
7등 이하 오류 처리 CHOOSE는 지정 밖이면 #VALUE! → IFERROR로 공백: IFERROR(CHOOSE(...),""). 또는 IF로: IF(RANK<=6, CHOOSE(...), "").
시험 예시: 등수→등급 매핑 + 나머지 공백. 두 방식(IFERROR / IF) 다 나옵니다.

📊 부분합 — 정렬 먼저, "새 값 대치" 해제, 윤곽 단추

1단계 정렬 그룹할 항목(진료 과목 등) 필드로 정렬을 먼저 합니다. 부분합은 자동 정렬을 하지 않아요(필기 단골 함정).
2단계 부분합 셀 포인터를 데이터 안에 두고 데이터 탭 → 개요 → 부분합. 그룹할 항목·사용 함수(합계/평균/최대/최소)·계산 항목 체크.
합계 위에 평균 함께 두 번째 부분합에서 "새로운 값으로 대치" 체크를 해제해야 앞의 합계가 유지됩니다(안 빼면 덮어써 사라짐). 순서 = 나중에 한 게 위. 합계는 "요약"으로 표시.
윤곽 단추 왼쪽 1=총합계만, 2=그룹 요약, 3=전체 펼침. 잘못 만들면 "모두 제거" 후 다시.
시험 예시: "몇 번 윤곽 단추를 눌렀나", "그룹할 항목 지정 시 자동 정렬되나(→아니오)"가 필기 단골.

👑 피벗 테이블 — 행·열·값, 값 필드 설정, 총합계

넣기 셀 포인터 데이터 안 → 삽입 탭 → 피벗 테이블(분석 도구 중 유일하게 데이터 탭이 아님). 행/열/값 영역에 끌어넣기(예: 부서=행, 직위=열, 실수령액=값).
합계↔평균 값은 기본 "합계". 값 필드 설정에서 평균·최대·최소 등으로 바꿉니다.
빈 셀·총합계 옵션에서 빈 셀 표시(하이픈 등). 열 총합계 / 행 총합계는 디자인 탭 → 총합계에서 남기거나 제거(무엇을 남겼는지 필기에 나옴). 값 2개↑면 Σ값을 행으로 옮겨 세로로.
시험 예시: 피벗은 시키는 게 많아 통째로 10점 — 부분 점수 없음. 차근차근 클릭.

🎨 조건부 서식 규칙 관리 + 홀짝 줄무늬(MOD·ROW)

수정 vs 삭제 규칙이 꼬이면 새 규칙을 자꾸 만들지 말고 규칙 관리에서 수정. 범위를 잘못 잡았으면 적용 대상을 다시 잡아 레코드만(삭제 후 재작성 금지).
홀짝 줄무늬 짝수 행만 색칠: 행 번호(ROW())를 2로 나눈 나머지가 0인지. =MOD(ROW(),2)=0. 참(TRUE)인 행만 채우기. 행 번호는 모든 열이 같아 열 절대 불필요.
시험 예시: 홀수/짝수 행 강조는 필기·실기 모두 출제. MOD·ROW 조합을 외웁니다.

🔁 (복습) AVERAGEIFS — 계산 범위가 맨 앞, 혼합참조

인수 순서 조건이 여러 개면 AVERAGEIFS(계산범위, 범위1, 조건1, 범위2, 조건2) — S가 붙으면 계산 범위가 맨 앞으로 옵니다.
혼합참조 부서×직위 표를 채우기 핸들 양방향으로 쓰면 혼합참조: 부서는 열 절대($부서), 직위는 행 절대(직위$). 계산범위는 F4 절대.
오류 해당 조합 데이터가 없으면 #DIV/0! → IFERROR로 공백. → 이 복잡한 걸 피벗이면 클릭 몇 번에 끝냅니다.
시험 예시: AVERAGEIFS·SUMIFS·COUNTIFS는 혼합참조가 핵심. 시험 아니면 안 쓰고 피벗으로 대체.

MATCH 검색범위에 필드명을 포함하면 행 번호가 한 칸씩 밀려 엉뚱한 값이 나옵니다.

검색범위는 머리글 빼고 레코드만, 위치를 맞추기.

자주 하는 실수

CHOOSE에서 7등 이하 #VALUE!를 방치해요

CHOOSE 지정 범위(1~6) 밖은 오류가 납니다.

IFERROR(CHOOSE(...),"") 또는 IF(RANK<=6, CHOOSE(...), "")로 공백.

자주 하는 실수

필기: exFAT를 윈도우 전용으로 착각해요

윈도우 전용은 NTFS입니다. exFAT는 오히려 이동식·호환용이에요.

NTFS=윈도우 전용 / exFAT=이동식·타 OS 호환으로 딱 붙여 외우기.

자주 하는 실수

필기: USB 파일 삭제가 복구된다고 생각해요

USB·이동식, Shift+Delete, 덮어쓰기는 휴지통에 안 가 복구가 안 됩니다.

이동식·Shift+Del·덮어쓰기 = 즉시 완전 삭제(그림 3).

자주 하는 실수

조건부 서식 규칙을 자꾸 새로 만들거나 지워요

규칙이 여러 개 쌓여 서로 꼬이고, 지웠다 다시 만들다 시간을 버립니다.

규칙 관리에서 수정, 범위는 적용 대상으로 다시 잡기(규칙은 하나만).
