🔮 가상 분석 3형제 📐 데이터 표=행/열 입력 셀 📊 부분합=정렬 먼저 👑 피벗=새로고침 수동 🧹 조각 모음=속도(용량 X) 🖨 스풀=효율 · 메모장=OLE 불가

📌오늘 배운 핵심 요약

실기

🔮 가상 분석 3형제 (예측 도구)

데이터 표=여러 변수를 표로 예측 / 시나리오=몇 가지 상황 가정 / 목표값 찾기=한 가지 결과가 되려면 입력값이 얼마?(역산). 모두 데이터 탭 → 가상 분석에 묶여 있어요.

예측할 개수: 데이터 표(많음) > 시나리오(2~3) > 목표값(1)
실기

📐 데이터 표 = 행 입력 셀 / 열 입력 셀

제일 헷갈리는 건 행/열 입력 셀 구분. 원래 수식에서 국어(행 데이터)는 행 입력 셀, 수학(열 데이터)은 열 입력 셀에 지정. 나머진 범위 잡고 확인이면 끝.

"몇 번째 행에 있나 / 몇 번째 열에 있나"로 구분
실기

🎯 목표값 찾기 = 결과 역산

"월 상환액이 2만원이 되려면 상환 기간은?" 처럼 결과를 정해놓고 입력값을 거꾸로 찾습니다. 수식 셀 / 찾는 값 / 값을 바꿀 셀 3개만.

한 개 셀만 바뀌는, 딱 한 가지 상황
실기

📊 부분합 = 정렬 먼저 + "새 값 대치" 해제

그룹할 항목으로 정렬이 먼저(자동 정렬 안 됨). 합계 위에 평균을 함께 남기려면 2번째에 "새로운 값으로 대치" 해제.

그룹 사이 페이지 나누기는 인쇄 구분(필기)
실기

👑 피벗 = 값 필드 설정·그룹·새로고침

값은 값 필드 설정으로 합계↔평균/최대/최소. 날짜는 그룹 필드(월 단위). 원본을 바꾸면 자동 반영 안 되고 "새로 고침" 필요.

그룹 필드는 그 열에 셀 포인터 둬야 활성화
필기

🧹 조각 모음 = 속도 개선(용량 X)

흩어진 조각을 한데 모아 읽기 속도를 개선. 용량이 늘어나는 게 아니에요. 하드 디스크만 하고, SSD는 조각 모음 대신 TRIM.

디스크 정리 대상이 아닌 것 = 글꼴
필기

🖨 스풀 = 효율(인쇄 속도 X) · 인쇄 단위

느린 프린터와 빠른 CPU를 병행 처리하려 하드 디스크에 임시 저장. 효율↑이지 인쇄 속도↑ 아님. 속도 단위 IPM=ISO 표준, DPI=해상도.

스풀하면 오히려 인쇄가 느려질 수도
필기

📝 메모장 = 서식 없는 텍스트

그림·표(OLE 개체) 삽입 불가, 색상 없음. 되는 건 페이지 설정·찾기바꾸기·F5(날짜/시간)·첫 줄 .LOG. 그림·문단 정렬은 워드패드.

확장자 .txt / 실행: Win+R → notepad

💡개념 다시 보기

🔮 가상 분석 3형제 — 데이터 표 · 시나리오 · 목표값 찾기 (오늘 신규)

공통 셋 다 "가상으로 미리 예측"하는 도구예요. 모두 데이터 탭 → 가상 분석 안에 묶여 있어 잘 못 찾습니다(리본에 안 펼쳐져 있음).
데이터 표 여러 변수를 표로 한꺼번에 예측(예: 국어·수학 점수가 바뀌면 합계가 어떻게?). 예측 개수가 가장 많아요.
시나리오 몇 가지 상황(2~3개)만 가정(예: 반품 수량이 줄면 vs 늘면 매출액 합계는?).
목표값 찾기 한 가지 결과를 정해놓고 입력값을 거꾸로 찾음(예: 월 상환액이 2만원이 되려면 상환 기간은?).
필기 예시: "몇 가지 상황을 가상으로 예측하는 기능은?" → 데이터 표 또는 시나리오. 둘 중 하나만 보기에 주어져요. "한 가지 결과의 역산"은 목표값 찾기.

📐 데이터 표 — 행 입력 셀 / 열 입력 셀 (가장 헷갈림)

준비 원본 수식(합계)값을 표 모서리로 당겨옵니다. 이때 그냥 붙여넣기 하면 채우기 핸들처럼 밀려요(상대 참조라 따라 내려옴). → F2로 수정 상태에서 드래그 복사하거나, 아예 셀 참조(=B5)로 당겨오기(제일 쉬움).
구분법 원래 수식에서 국어 점수가 10행에 있으면 "행 데이터" → 행 입력 셀, 수학 점수가 B열에 있으면 "열 데이터" → 열 입력 셀.
실행 표 범위 선택 → 가상 분석 → 데이터 표 → 행 입력 셀·열 입력 셀만 지정하면 5초면 끝. 확인 버튼이면 완료.
시험 예시: 행/열 입력 셀 구분만 정확하면 아주 쉬운 문제. 범위 선택 후 대화상자를 여는 순서를 헷갈려 어려워합니다.

🎬 시나리오 — 이름 정의 · 변경 셀 · 결과 셀 · 요약

이름 정의 먼저 요약 결과가 셀 주소(예: $B$4) 대신 이름으로 나오게, 변경 셀·결과 셀에 이름 지정(이름 상자에 입력). 오타는 수식 탭 → 이름 관리자에서 편집.
시나리오 관리자 데이터 탭 → 가상 분석 → 시나리오 관리자 → 추가로 극본 만들기(예: "수익률 증가"=반품 수량 줄임, "수익률 감소"=늘림). 변경 셀에 상황별 숫자 입력.
요약 제대로 만들면 추가·요약 두 개만 쓰면 끝. 결과 셀=변동을 보고 싶은 값(매출액 합계). 시나리오 요약 시트 위치까지 문제에 나옵니다.
시험 예시: "변경 셀 / 결과 셀"을 헷갈려요. 바꾸는 게 변경 셀, 결과로 보는 게 결과 셀(매출액 합계).

🎯 목표값 찾기 — 수식 셀 · 찾는 값 · 값을 바꿀 셀

수식 셀 내가 특정 값이 되길 바라는 셀(예: 월 상환액).
찾는 값 그 셀이 되어야 할 값(예: 20000).
값을 바꿀 셀 엑셀이 거꾸로 계산해 바꿔 줄 셀(예: 상환 기간). 내가 못 구하니 엑셀에게 맡기는 거예요. 확인이면 끝.
시험 예시: "합계가 270점이 되려면 국어 점수는?" 같은 한 가지 상황의 역산이 목표값 찾기.

📊 부분합 — 정렬 먼저, "새 값 대치" 해제, 그룹 사이 페이지 나누기

1단계 정렬 그룹할 항목(진료 과목 등)으로 정렬을 먼저. 부분합은 자동 정렬을 하지 않습니다(필기 단골).
2단계 부분합 데이터 탭 → 개요 → 부분합. 그룹할 항목·사용 함수(합계/평균)·계산 항목 체크.
합계 위에 평균 함께 두 번째 부분합에서 "새로운 값으로 대치" 해제(안 빼면 앞 합계가 사라짐). 나중에 한 게 위로 올라옵니다.
그룹 사이 페이지 나누기(필기) 체크하면 그룹마다 인쇄 페이지가 나뉘도록 선이 들어갑니다. 실기엔 안 나오고 필기에 나와요.
시험 예시: "그룹할 항목 지정 시 자동 정렬되나?(→아니오)", "그룹 사이 페이지 나누기의 효과"가 필기 포인트.

👑 피벗 테이블 — 값 필드 설정 · 그룹 · 새로 고침 · 레이아웃

넣기 셀 포인터 데이터 안 → 삽입 탭 → 피벗 테이블(분석 도구 중 유일하게 삽입 탭). 기존 워크시트에 위치 지정(예: I3). 필드를 행/열/값/필터로 끌어넣기.
값 필드 설정 기본 "합계" → 평균·최대·최소로. 표시 형식(천 단위 구분)은 값 필드 설정 → 셀 서식에서 값마다 따로.
그룹 · 새로 고침 날짜를 월 단위로 묶으려면 그룹 필드(그 열에 셀 포인터 둬야 활성화). 원본 데이터를 바꾸면 자동 반영 안 됨 → "새로 고침"(필기 단골). 정렬은 필터 단추로.
레이아웃 디자인 탭 → 보고서 레이아웃(개요/테이블 형식), 행/열 총합계 표시·해제, 피벗 스타일. 값 2개↑면 Σ값을 행으로 옮겨 세로로.
시험 예시: "원본을 고치면 피벗이 자동 갱신되나?(→아니오, 새로 고침)", "피벗 시작 위치는 I1?(→아니오, 지정한 I3)".

🔁 (복습) 1교시 함수보충2 — 오늘의 실기 복습

SUM 누적 누적 판매량은 =SUM($B$41:B41) — 앞 셀만 절대 참조, 뒤는 상대로 두고 채우기 핸들. 한 칸씩 늘어나며 누적됩니다.
SMALL + 시·분·초 가장 빠른 기록은 SMALL(범위,1)로 최소를 찾고, HOUR/MINUTE/SECOND로 각각 분해(반복은 Ctrl+C/V).
다시 복습 TIME(+2분)·WEEKDAY 중첩 IF(반납일)·DMAX→MATCH→INDEX·RIGHT×1 숫자화는 11차시에서 배운 패턴 그대로예요.
1교시는 대부분 어제(11차시) 패턴의 응용 문제. 안 되는 패턴만 적어두고 반복 연습하세요.

🖥필기 그림으로 이해하기 (컴퓨터 일반)

① 가상 분석 3형제 한눈에 (실기)

"많이 예측 → 몇 개 → 딱 하나" 순서로 데이터 표 · 시나리오 · 목표값 찾기를 외우면 쉬워요.

② 드라이브 조각 모음 원리 (속도 개선 = 용량 아님)

하드 디스크는 흩어진 조각을 이리저리 읽어 느려져요. 한데 모으면 회전하며 순서대로 읽어 빨라집니다.

③ 디스크 정리 대상 O / X

디스크 정리는 불필요한 파일을 지워 저장 공간을 늘려요. 그런데 글꼴은 지우지 않습니다.

④ 프린터 스풀(SPOOL) 개념

빠른 CPU가 느린 프린터를 붙잡고 있으면 비효율. 인쇄 내용을 하드 디스크에 잠깐 저장하고 CPU는 다른 일을 합니다.

⑤ 인쇄 속도·해상도 단위

"무엇을 1분(초)에 몇 개?"로 외우면 쉬워요. IPM은 ISO 국제표준, DPI는 해상도입니다.

⑥ 메모장 vs 워드패드

둘 다 윈도우 기본 유틸리티지만, 그림·표(OLE)·문단 정렬에서 갈립니다.

🚨자주 하는 실수

⭐ 가장 많이 헷갈림

데이터 표에서 행 입력 셀 / 열 입력 셀을 바꿔 넣어요

원래 수식에서 국어가 몇 번째 행에 있으면 행 입력 셀, 수학이 몇 번째 열에 있으면 열 입력 셀. 이걸 반대로 넣으면 값이 다 틀립니다.

"국어=행 데이터 → 행 입력 셀 / 수학=열 데이터 → 열 입력 셀"로 딱 붙여 외우기.

자주 하는 실수

데이터 표 값을 그냥 붙여넣기 해요

원본 수식값을 통째로 복사·붙여넣기 하면 상대 참조라 채우기 핸들처럼 밀려서 들어옵니다.

F2로 수정 상태에서 드래그 복사하거나, 아예 셀 참조(=B5)로 당겨오기(제일 쉬움).

자주 하는 실수

부분합 전에 정렬을 안 해요

부분합은 그룹할 항목을 지정해도 자동으로 정렬하지 않습니다. 정렬 안 하면 그룹이 흩어져요.

그룹할 항목 필드로 정렬을 먼저 한 뒤 부분합.

자주 하는 실수

피벗 원본을 고치고 "새로 고침"을 안 해요

원본 데이터를 바꿔도 피벗은 자동으로 갱신되지 않습니다. 옛날 값 그대로예요.

피벗 안에 셀 포인터 → 새로 고침(데이터 탭). 원본 변경은 수동 업데이트.

자주 하는 실수

피벗 그룹 필드가 회색(비활성)이라 못 눌러요

월 단위로 묶으려는데 "그룹 필드"가 활성화 안 되는 건, 셀 포인터가 그룹할 필드에 없기 때문이에요.

묶을 필드(날짜 열)에 셀 포인터를 두고 그룹 필드를 누르기.

자주 하는 실수

필기: 조각 모음이 "용량"을 늘린다고 착각해요

조각 모음(최적화)은 흩어진 조각을 모아 읽기 속도를 높이는 것이지, 저장 용량이 늘어나는 게 아닙니다.

조각 모음 = 속도 개선 / 공간 확보는 디스크 정리·휴지통 비우기·압축.

자주 하는 실수

필기: 글꼴을 디스크 정리 대상이라고 생각해요

디스크 정리는 휴지통·임시·캐시·다운로드 파일을 지우지만, 글꼴(폰트)은 대상이 아니에요.

정리 대상이 아닌 것 = 글꼴(그림 3).

자주 하는 실수

필기: "빠른 포맷"이면 완전히 지워진다고 생각해요

빠른 포맷은 링크만 끊어 데이터가 남아 복원될 수 있어요(중고로 팔면 위험).

완전히 지우려면 빠른 포맷 체크 해제 후 일반 포맷(속도는 느리지만 안전).

자주 하는 실수

필기: 스풀이 "인쇄 속도"를 높인다고 생각해요

스풀은 CPU와 프린터를 병행 처리해 효율을 높이는 것. 인쇄 속도가 빨라지는 건 아니며 오히려 느려질 수도 있어요.

스풀 = 효율↑(병행 처리) / 인쇄 속도↑ 아님(그림 4).

자주 하는 실수

필기: 메모장에 그림·표를 넣을 수 있다고 생각해요

메모장은 서식 없는 텍스트라 그림·표(OLE 개체) 삽입이 안 됩니다.

그림·문단 정렬이 필요하면 워드패드(그림 6). 메모장은 F5 날짜·.LOG는 됨.
