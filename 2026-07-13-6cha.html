참조 복습(상대·절대·혼합) SUMIF·SUMIFS·DSUM 계열 · ROUND·MOD

오늘의 핵심은 "함수는 인수 순서를 정확히 외우고, 범위는 절대 참조로 고정하는 것"입니다.

📌오늘 배운 핵심 요약

함수 문제는 대부분 중첩으로 출제된다

실기 함수 문제는 함수 하나만 쓰는 경우보다 여러 함수를 겹쳐서 쓰는 경우가 훨씬 많습니다. 그래서 개별 함수의 역할과 인수 순서를 정확히 외워두는 게 먼저입니다.

지금 입력 중인 게 몇 번째 인수인지 항상 확인하는 습관

참조 복습: 상대·절대·혼합, F4

상대 참조는 채우기 핸들 방향에 따라 주소가 바뀌고, 절대 참조는 $로 행·열을 고정합니다. 혼합 참조는 수식을 가로·세로 양방향으로 채울 때 필요합니다.

F4를 누르면 상대→절대→행고정→열고정 순으로 순환

SUMIF·SUMIFS: 조건부 합계

SUMIF는 조건 1개 기준 합계(조건범위, 조건, 합계범위), SUMIFS는 여러 조건 기준 합계이며 합계범위가 맨 앞에 옵니다. 순서가 다르다는 점이 헷갈리기 쉽습니다.

SUMIF는 "찾고→합친다", SUMIFS는 "합칠 것부터"

데이터베이스 함수: DSUM·DAVERAGE·DCOUNT·DCOUNTA

데이터베이스, 필드, 조건 목록을 인수로 씁니다. 조건 목록은 고급 필터처럼 별도 범위로 만들고, 같은 줄은 AND, 다른 줄은 OR로 해석됩니다.

필드는 선택한 범위 기준 몇 번째 열인지로 계산

ROUND 계열: 반올림·올림·내림

ROUND(반올림), ROUNDUP(올림), ROUNDDOWN(내림)은 모두 자릿수 인수로 어느 자리까지 처리할지 정합니다. 자릿수가 양수/0/음수일 때 의미가 다릅니다.

자릿수 양수=소수점 아래, 0=일의 자리, 음수=정수 자리

MOD 함수로 나머지 계산·그룹 나누기

MOD는 나눗셈의 나머지를 구하는 함수로, 번호를 일정 그룹으로 나누거나 짝수·홀수를 구분하는 문제, 주민등록번호에서 성별을 판별하는 중첩 문제에 자주 쓰입니다.

MID로 문자 추출 → MOD로 판별 → IF로 결과 표시, 3단 흐름

참조 복습 — 상대·절대·혼합, F4로 순환

상대 참조 C7처럼 그냥 쓰면 채우기 핸들로 복사할 때 주소가 따라 움직입니다. 아래로 내리면 행 번호가, 오른쪽으로 끌면 열 문자가 바뀝니다.
절대 참조 $C$7 — 열과 행 모두 고정. 어디로 복사해도 항상 같은 셀을 봅니다.
혼합 참조 $B8은 열만 고정(B는 안 변함), C$7은 행만 고정(7은 안 변함). 수식을 가로·세로 양방향으로 채울 때 필요합니다.
F4 키 주소를 입력한 뒤 F4를 누르면 C7 → $C$7 → C$7 → $C7 순으로 순환됩니다.
실전 원칙 SUMIF·SUMIFS·DSUM처럼 "모든 행이 같은 범위를 봐야 하는 것"은 절대 참조로 고정하고, 조건처럼 행마다 달라져야 하는 것은 상대 참조로 둡니다.
시험 예시: 채우기 핸들로 아래로 복사했더니 결과가 이상하다 → 범위가 절대 참조로 고정되지 않아 함께 밀려 내려간 경우가 대부분입니다.

SUMIF·SUMIFS — 인수 순서가 서로 다르다

SUMIF 구조 SUMIF(조건범위, 조건, [합계범위]) — 조건 범위와 합계 범위가 같으면 합계 범위를 생략할 수 있습니다.
SUMIFS 구조 SUMIFS(합계범위, 조건범위1, 조건1, 조건범위2, 조건2, …) — SUMIF와 달리 합계 범위가 맨 앞에 옵니다.
비교 조건 작성 "40만원 이상"처럼 비교 연산자가 들어가는 조건은 반드시 따옴표로 묶어 ">=400000" 형태로 씁니다.
범위 실수 방지 range·criteria처럼 범위를 뜻하는 인수 자리에는 실제 셀 범위를, 조건 자리에는 조건 값을 넣어야 합니다. 자리를 바꿔 넣는 실수가 잦습니다.
시험 예시: "SUMIF의 세 번째 인수(합계 범위)를 생략할 수 있는 경우"와 "SUMIFS는 합계 범위가 가장 먼저 온다"는 점을 헷갈리지 않도록 구분해서 외우세요.

데이터베이스 함수 — DSUM·DAVERAGE·DCOUNT·DCOUNTA

기본 구조 함수명(database, field, criteria) — 데이터베이스 범위, 필드(몇 번째 열인지), 조건 목록 순서입니다.
조건 목록 고급 필터처럼 필드명과 조건을 별도 셀 범위로 미리 만들어야 합니다. 조건을 같은 줄에 쓰면 AND, 다른 줄에 쓰면 OR입니다.
필드 번호 주의 필드는 선택한 데이터베이스 범위를 기준으로 몇 번째 열인지 계산합니다. 전체 표를 선택했을 때와 일부 열만 선택했을 때 번호가 달라질 수 있습니다.
DCOUNT vs DCOUNTA DCOUNT는 숫자가 들어 있는 필드의 개수만 세고, DCOUNTA는 문자든 숫자든 값이 있는 필드를 모두 셉니다.
시험 예시: 조건 목록의 필드명이 원본 표와 한 글자라도 다르면 조건이 무시됩니다. 고급 필터와 같은 원리입니다.

ROUND 계열 — 반올림·올림·내림

ROUND 지정한 자릿수에서 반올림합니다.
ROUNDUP 지정한 자릿수에서 무조건 올림합니다.
ROUNDDOWN 지정한 자릿수에서 무조건 내림(버림)합니다.
자릿수 인수 규칙 양수면 소수점 아래 자리, 0이면 일의 자리(정수)까지, 음수면 십의 자리·백의 자리 등 정수 자리를 처리합니다.
시험 예시: 자릿수에 음수를 넣는 문제(예: 백의 자리에서 반올림)가 종종 나오니 부호에 따른 의미를 정확히 기억하세요.

MOD 함수와 중첩 함수 실전

MOD란 나눗셈을 했을 때 나머지를 구하는 함수입니다. 번호를 그룹으로 나누거나 짝수·홀수를 구분할 때 활용합니다.
MID + MOD + IF 조합 주민등록번호 등 문자열에서 MID로 특정 위치 문자를 추출한 뒤, MOD로 홀짝을 판별하고, IF로 "남"/"여" 같은 결과를 표시하는 흐름으로 연결합니다.
고객 ID별 합계 특정 고객의 전체 주문액처럼 열 전체 범위를 대상으로 합계를 구할 때는 SUMIF·SUMIFS 같은 조건부 합계 함수를 사용합니다.
시험 예시: 중첩 함수는 안쪽 함수부터 순서대로 실행됩니다. 가장 안쪽 함수의 결과가 무엇인지부터 확인하고 바깥쪽 함수로 넘어가세요.

범위를 지정한 직후 F4를 눌러 $를 붙이는 습관을 들이세요.

자주 하는 실수

"40만원 이상"처럼 비교 연산자가 들어간 조건은 ">=400000"처럼 따옴표로 묶어야 하는데, 이를 빠뜨리면 조건이 제대로 해석되지 않을 수 있습니다.

비교 연산자가 들어가면 무조건 큰따옴표로 묶는다고 외우세요.

데이터베이스 함수의 필드 번호를 잘못 계산해요

선택한 범위가 전체 표냐 일부 열이냐에 따라 같은 필드라도 몇 번째 열인지가 달라지는데, 이를 확인하지 않고 번호를 넣는 실수가 많습니다.

필드 번호를 넣기 전, 내가 선택한 database 범위의 첫 번째 열부터 순서대로 세어보세요.

필드가 숫자 데이터인지 문자 데이터인지 먼저 확인하고 함수를 고르세요.
