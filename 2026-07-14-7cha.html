찾기·참조 함수 VLOOKUP·HLOOKUP 텍스트 함수 · 연결 연산자(&)

오늘의 핵심은 "찾을 값을 기준표 첫 열(행)에 맞추고, 기준표는 절대 참조로 고정하는 것"입니다.

🔎 VLOOKUP (세로) ↔️ HLOOKUP (가로) ✂️ LEFT·MID·RIGHT 🔡 UPPER·LOWER·PROPER 🧹 TRIM·FIND·SEARCH 🔗 연결 연산자 &

📌오늘 배운 핵심 요약

먼저 DB함수·SUMIF 복습부터

수업 앞부분은 어제 배운 DSUM·DAVERAGE·DCOUNTA와 SUMIF·SUMIFS·COUNTIF 복습이었습니다. 조건을 뺀 평균은 AVERAGE로 한 번에 말고 SUMIF÷COUNTIF로 나눠 구하는 패턴도 다시 짚었습니다.

0을 뺀 평균: 합계도 개수도 같은 조건을 걸어야 함

VLOOKUP = 세로로 찾기

VLOOKUP(찾을값, 기준표, 열번호, 일치옵션). 기준표의 첫 번째 열에서 찾을값을 세로로 찾아, 같은 행의 지정한 열 값을 가져옵니다. 세로로 길게 쌓인 표에 씁니다.

V = Vertical(수직) → 첫 "열"에서 찾는다

HLOOKUP = 가로로 찾기

HLOOKUP(찾을값, 기준표, 행번호, 일치옵션). 기준표의 첫 번째 행에서 찾을값을 가로로 찾아, 같은 열의 지정한 행 값을 가져옵니다. 가로로 눕힌 표에 씁니다.

H = Horizontal(수평) → 첫 "행"에서 찾는다

일치 옵션: FALSE vs TRUE

이름·코드처럼 딱 맞는 값을 찾을 땐 FALSE(정확히 일치). 점수 구간(90 이상 A 등)처럼 "이상~미만"으로 찾을 땐 TRUE(유사 일치). TRUE는 기준표가 오름차순 정렬돼 있어야 합니다.

문자·코드=FALSE / 구간·등급=TRUE

텍스트 자르기: LEFT·MID·RIGHT

LEFT(왼쪽부터 n글자), RIGHT(오른쪽부터 n글자), MID(시작위치부터 n글자). MID의 세 번째 인수는 "끝 위치"가 아니라 가져올 글자 수입니다.

LEN(글자 수), UPPER/LOWER/PROPER(대·소·첫글자 대문자)

다듬기·찾기·연결: TRIM·FIND·&

TRIM은 불필요한 공백 제거(lookup 매칭 안 될 때 특효), FIND/SEARCH는 특정 문자 위치 찾기, &는 문자·함수 결과를 이어붙이는 연결 연산자입니다.

기호·단위 표시는 "따옴표"로 묶어 &로 연결

💡개념 다시 보기

VLOOKUP — 세로 기준표에서 찾기

구조 =VLOOKUP(찾을값, 기준표범위, 열번호, 일치옵션). 예: =VLOOKUP(B2,$F$2:$H$10,3,FALSE)
1인수 찾을값 기준표에서 찾을 값(코드·이름 등). 보통 현재 시트의 셀을 가리킵니다.
2인수 기준표 찾을값이 반드시 맨 왼쪽 열에 오도록 범위를 잡습니다. 채우기 핸들로 복사할 것이므로 F4로 절대 참조($) 고정.
3인수 열번호 기준표 왼쪽에서 몇 번째 열의 값을 가져올지. 기준표 첫 열이 1번입니다(전체 시트 열이 아님).
4인수 일치옵션 FALSE(정확히 일치) / TRUE(유사 일치). 이름·코드는 FALSE.
시험 예시: 사원표에서 사번으로 이름·부서를 가져오기. 열번호만 3→4로 바꿔 여러 열을 뽑습니다.

HLOOKUP — 가로 기준표에서 찾기

구조 =HLOOKUP(찾을값, 기준표범위, 행번호, 일치옵션).
차이점 VLOOKUP은 첫 "열"에서 세로로 찾고, HLOOKUP은 첫 "행"에서 가로로 찾습니다. 기준표가 눕혀진(가로로 긴) 형태면 HLOOKUP.
3인수 행번호 기준표 위에서 몇 번째 행의 값을 가져올지.
판단법 기준표의 방향(찾을 대상이 첫 열에 세로로? 첫 행에 가로로?)만 보면 V인지 H인지 바로 정해집니다.
시험 예시: 표의 머리글(첫 행)이 등급 A·B·C처럼 가로로 나열돼 있으면 HLOOKUP.

일치 옵션 FALSE / TRUE, 그리고 #N/A

FALSE(0) 정확히 같은 값만 찾음. 이름·코드·부서명 등 문자 검색은 FALSE. 못 찾으면 #N/A.
TRUE(1) 또는 생략 유사 일치. "이 값 이하 중 가장 큰 값"을 찾음. 점수→등급, 판매액→인센티브율처럼 구간 기준일 때. 단, 기준표가 오름차순 정렬돼 있어야 정확합니다.
#N/A 오류 점검 순서 ①찾을값이 기준표 첫 열(행)에 실제로 있는가 ②기준표 범위가 맞는가 ③열/행 번호가 범위 안인가 ④공백·표시형식 차이는 없는가.
시험 예시: 문자 검색에 TRUE(또는 옵션 생략)를 쓰면 엉뚱한 값이 나옵니다 — 문자는 반드시 FALSE.

텍스트 함수 — 자르고·바꾸고·다듬기

LEFT / RIGHT 왼쪽/오른쪽 끝에서 n글자. 예: =LEFT(A2,2) 코드 앞 2자리, =RIGHT(A2,4) 뒤 4자리.
MID =MID(문자열, 시작위치, 글자수). 세 번째 인수는 "몇 번째까지"가 아니라 몇 글자를 가져올지입니다. 주민번호 뒤 성별자리 추출 등.
LEN 문자열의 글자 수를 셉니다.
UPPER / LOWER / PROPER 모두 대문자 / 모두 소문자 / 각 단어 첫 글자만 대문자.
TRIM 앞뒤·중복 공백 제거. lookup이 값은 같은데 매칭 안 될 때 공백 때문인 경우가 많아 TRIM으로 해결합니다.
FIND / SEARCH 특정 문자의 위치를 반환. FIND는 대소문자 구분, SEARCH는 구분 안 함·와일드카드 가능. 예: =LEFT(A2,FIND("@",A2)-1) 이메일 아이디만 추출.
시험 예시: =FIND(",",A2)-1처럼 위치에서 1을 빼서 구분자 앞부분만 잘라냅니다.

연결 연산자 & 와 함수 중첩

& 연결 문자·셀·함수 결과를 이어붙입니다. 예: =B2&" "&C2 (성+이름), ="20"&LEFT(A2,2)&"년".
기호·단위 표시 괄호·하이픈·"명"·"년" 같은 문자는 따옴표로 묶어 &로 연결해야 화면에 표시됩니다.
검색값 만들기 부서&직급처럼 두 값을 &로 합쳐 하나의 검색 키를 만들고, 그걸 VLOOKUP 찾을값으로 넣기도 합니다.
중첩 순서 복잡한 중첩은 안쪽 함수부터 실행됩니다. LEFT/MID/RIGHT로 먼저 값을 만들고, 그 결과를 VLOOKUP·HLOOKUP의 찾을값으로 감쌉니다.
시험 예시: =VLOOKUP(LEFT(A2,2),$F$2:$G$8,2,FALSE) — 코드 앞 2자리로 부서명 찾기.

🚨자주 하는 실수

⭐ 가장 많이 틀린 부분

VLOOKUP 기준표에 머리글을 넣거나, 찾을값이 첫 열에 없어요

찾을 값은 반드시 기준표 맨 왼쪽 열(HLOOKUP은 첫 행)에 있어야 합니다. 머리글까지 범위에 넣거나 찾을값 열이 가운데 있으면 잘못된 값이나 #N/A가 납니다.

기준표 범위를 잡을 때 "찾을 대상이 이 범위의 맨 앞 열(행)인가?"를 먼저 확인하세요.

자주 하는 실수

기준표 범위를 절대 참조로 고정 안 해요

채우기 핸들로 아래로 복사하면 기준표 범위가 같이 밀려 내려가 아래 행부터 결과가 틀어집니다.

기준표 범위를 지정한 직후 F4를 눌러 $F$2:$H$10처럼 절대 참조로 고정하세요.

자주 하는 실수

문자 검색인데 FALSE를 안 써요

이름·코드 같은 문자를 찾을 때 일치 옵션을 TRUE로 두거나 생략하면 엉뚱한 값이 나옵니다.

정확히 일치시켜야 하는 문자·코드 검색은 무조건 마지막 인수에 FALSE(또는 0).

자주 하는 실수

MID의 세 번째 인수를 "끝 위치"로 착각해요

MID(문자열, 시작위치, 글자수)인데 세 번째를 끝 위치로 넣어 엉뚱한 길이가 잘립니다.

"시작 위치부터 몇 글자" — 세 번째는 개수라고 외우세요.

자주 하는 실수

값은 같은데 lookup 매칭이 안 돼요 (공백·표시형식)

눈으로는 같아 보여도 앞뒤 공백이나 텍스트 표시형식 때문에 다른 값으로 인식돼 #N/A가 납니다.

TRIM으로 공백을 제거하거나, 표시 형식을 '일반'으로 바꿔 확인하세요.

자주 하는 실수

0을 뺀 평균에서 개수를 잘못 세요

SUMIF로 0이 아닌 값만 더해 놓고, 나눌 때 COUNT로 0까지 포함해 세면 평균이 틀립니다.

합계와 개수 둘 다 "0이 아닌"("<>0") 같은 동일 조건을 걸어야 합니다.
