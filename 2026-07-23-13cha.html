🔢 자료 표현·저장 단위, 진법 (필기)

표현 단위 bit(0·1, 정보 최소 단위) → 니블(4bit, 16진수 표현) → byte(8bit, 문자 1개, 256가지=0~255) → 워드(CPU가 한 번에 처리하는 명령어 단위, 기본 2byte, 32/64bit) → 필드 → 레코드 → 파일 → 데이터베이스.
저장 용량 KB < MB < GB < TB … 2¹⁰=1024배씩 커집니다.
진법 10↔2진은 2로 나눈 나머지를 거꾸로(=MOD). 2진→8진은 3bit씩(가중치 1·2·4), 2진→16진은 4bit씩(가중치 1·2·4·8). 자리값 1·2·4·8을 외우고, 16진수 10~15는 A~F.
시험 예시: "1byte로 표현 가능한 정보 수 = 256(2⁸)", "워드=CPU 명령어 처리 단위" 단골.

🔤 문자 코드 & 에러 검출 (빨간 별표 암기)

BCD 6bit — 영문 소문자 표현 불가.
아스키(ASCII) 7bit(확장형 8bit) — 데이터 통신·개인용 PC.
EBCDIC 8bit — 대형(메인프레임) 컴퓨터.
유니코드 16bit=2byte — 전 세계 문자 65536자, 한글 완성형+조합형 모두 표현.
에러 검출 패리티 비트=오류 검출만 / 해밍 코드=오류 검출+교정. ⚠️ 패리티 비트는 "문자 표현 코드"가 아님(함정).
시험 예시: 순서 6·7·8·16으로 외우면 bit 수 문제를 바로 풉니다.

🖥 컴퓨터 분류 & 자료 처리 방식 (필기)

데이터별 디지털(0·1·산술논리·정밀·범용) / 아날로그(연속 물리량·미적분·빠름·특수목적) / 하이브리드(둘의 장점 결합).
처리 능력별 슈퍼 > 메인프레임 > 미니 > 워크스테이션 > 마이크로(우리가 쓰는 것, 웨어러블 포함).
자료 처리 일괄(온·오프라인 둘 다)·실시간(예약·은행)·시분할(1대를 여럿이 시간 쪼개 사용)·분산(떨어진 컴퓨터 연결).
이중화 듀얼=2대 동시 가동 / 듀플렉스=1대만 가동, 나머지는 고장 대비 대기. 클라이언트/서버 vs P2P(서버·클라이언트 동시, 바이러스 취약).
시험 예시: 자료(raw)를 가공하면 정보. 듀얼/듀플렉스 구분, P2P의 보안 취약점이 단골.

🧠 CPU 구조 & 기억장치 (필기 — 오늘 꼭 외우기)

CPU 구성 제어장치 + 연산장치(ALU) + 레지스터.
제어장치 PC(프로그램 카운터)=다음에 실행할 명령어의 주소 기억 / IR(명령 레지스터)=현재 실행 중인 명령어 기억 / 디코더·인코더=해독·부호화.
연산장치(ALU) AC(누산기)=연산 중간 결과 임시 저장 / 상태 레지스터=오버/언더플로우·캐리 / 가산기=덧셈 / 보수기=뺄셈(보수를 취해 덧셈으로 처리).
기억장치 속도 레지스터 > 캐시(SRAM) > 주기억(DRAM=램) > 보조(하드). 캐시 메모리=CPU·주기억 속도 차를 줄이는 고속 완충(SRAM). 가상 메모리=보조기억 일부를 주기억처럼 사용(큰 프로그램 실행 O, 속도 느려짐 단점). 플래시=EEPROM. ROM=비휘발성(POST) / RAM=휘발성.
시험 예시: PC/IR/누산기/보수기 역할, 캐시=SRAM, 가상 메모리 정의가 계속 나옵니다.

기록 중지를 안 하면 이후 도형 그리기까지 기록돼, 매크로 실행 때마다 도형이 계속 생깁니다.

동작(첫 줄)이 끝나면 바로 기록 중지. 도형 그리기 전에 반드시 중지.

자주 하는 실수

기본 xlsx로 저장하면 매크로 코드가 저장되지 않아요.

xlsm(매크로 사용 통합 문서)로 저장.

기록 전 셀 포인터를 데이터 바깥으로 옮겨, 셀 선택까지 기록되게.

필기 · 자주 틀림

워드(word)와 바이트(byte)를 헷갈려요

문자 1개는 byte(8bit), CPU가 한 번에 처리하는 명령어 단위는 워드입니다.

byte=문자 1개, 워드=CPU 명령어 처리 단위(32/64bit).

필기 · 자주 틀림

유니코드의 bit 수를 잘못 알아요

BCD 6 / 아스키 7 / EBCDIC 8 / 유니코드 16bit=2byte. 유니코드가 전 세계 문자를 표현합니다.

순서 6·7·8·16으로 외우기. 유니코드=2byte.

필기 · 자주 틀림

디지털과 아날로그 특징을 반대로 골라요

미적분 연산·연속 물리량은 아날로그, 산술논리 연산·0·1은 디지털이에요.

아날로그=연속·미적분·특수목적 / 디지털=이산·산술논리·범용.

필기 · 자주 틀림

보수기를 "지수 표현"이라고 착각해요

보수기는 보수를 취해 덧셈으로 처리하는 뺄셈 장치입니다.

가산기=덧셈, 보수기=뺄셈. 누산기=중간 결과 임시 저장.

필기 · 자주 틀림

캐시 메모리와 가상 메모리를 헷갈려요

캐시=CPU와 주기억 속도차를 줄이는 고속 완충(SRAM), 가상 메모리=보조기억을 주기억처럼 빌려 쓰는 것.

캐시=SRAM·속도 완충 / 가상 메모리=보조기억 일부를 주기억처럼.
