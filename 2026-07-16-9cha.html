📌오늘 배운 핵심 요약

CHOOSE — 숫자로 고른다

=CHOOSE(숫자, 값1, 값2, …). 첫 인수에는 반드시 숫자가 들어가고 1번부터 시작합니다(0번 없음). 코드가 0부터면 +1로 맞춰줘야 해요.

대화상자(fx)에서 Tab키 = 따옴표·콤마 자동

RANK.EQ — 순위 매기기

=RANK.EQ(내 점수, 전체 범위, 옵션). 내 점수가 먼저, 전체 범위는 F4로 절대 고정. 옵션 0=내림차순, 1=오름차순이고 생략하면 내림차순입니다.

점수·금액=내림차순 / 기록·등급=오름차순(1)

SMALL·LARGE·MEDIAN·MODE

SMALL(범위,k)=k번째로 작은 값, LARGE(범위,k)=k번째로 큰 값. MEDIAN=중앙값, MODE.SNGL=최빈값(없으면 #N/A).

k=1이면 최저/최고 · 총점은 범위에서 빼기

IF — 조건식은 "질문"이다

=IF(조건식, 참일때, 거짓일때). 조건식은 엑셀에게 던지는 질문이고, 답은 TRUE/FALSE 둘뿐. 답이 나오면 해당 값을 출력하고 즉시 종료합니다.

IF 조건식엔 따옴표 없음 (SUMIF 조건과 반대)

중첩 IF — 구간 N개면 IF N-1개

결과가 A·B·C·D·F 5개면 질문은 4번이면 됩니다(마지막은 자동으로 거짓 구간). 거짓 구간 안에 다음 질문을 쌓아 올리는 구조예요.

먼저 구간 수를 세고 → IF 개수를 정하고 → 식을 쓴다

AND·OR — 질문을 동시에

IF 조건식엔 질문을 1개만 넣을 수 있어요. 여러 질문이 동시에 필요하면 조건식 안에 AND(모두 만족) 또는 OR(하나만)를 씁니다.

AND·OR의 괄호는 조건식 안에서 닫는다

날짜·시간 함수

TODAY()·NOW()는 인수가 없습니다. YEAR/MONTH/DAY로 쪼개고 DATE로 합치고, HOUR/MINUTE/SECOND로 쪼개고 TIME으로 합칩니다.

Ctrl+; = 오늘 날짜 / Ctrl+Shift+; = 현재 시간

WEEKDAY ★ 시험 단골

=WEEKDAY(날짜, 옵션). 옵션 2를 쓰면 월요일이 1이라 평일·주말 구분이 쉽습니다. CHOOSE와 겹쳐 한글 요일을, IF와 겹쳐 평일/주말을 판별해요.

옵션 2에서 평일은 5 이하

CHOOSE — 숫자에 맞는 값을 고른다

구조 =CHOOSE(숫자, 값1, 값2, …). 예: =CHOOSE(D4,"사원","대리","과장","차장","부장")
첫 인수는 반드시 숫자 문자를 넣으면 안 됩니다. 그리고 1번부터 시작하며 0번은 없습니다. 코드가 0,1,2…로 시작하면 +1을 해서 1번부터 되게 맞춰야 합니다.
대화상자(fx)를 쓰세요 값 칸에서 Tab키를 누르면 따옴표와 콤마를 자동으로 붙여줍니다. 직접 "사원", 치는 것보다 훨씬 빠르고 실수가 적습니다. (Tab=다음 칸, Shift+Tab=이전 칸)
기준표를 셀로 참조할 때는 F4 사원·대리·과장 목록을 따로 만들어 셀로 참조하면 나중에 직급 체계가 바뀌어도 그 표만 고치면 됩니다. 단 채우기 핸들을 쓰면 밀리므로 범위가 아닌 단일 셀이라도 "기준값"이면 절대 참조합니다.
시험 예시: VLOOKUP으로도 되는 문제를 CHOOSE로 풀게 하는 유형이 나옵니다. 첫 인수 자리에 RANK 같은 함수를 끼워 넣는 중첩도 잘 나옵니다.

SWITCH — CHOOSE와 닮았지만 문자도 된다

차이 CHOOSE는 첫 인수가 반드시 숫자이고 1,2,3… 순서가 고정입니다. SWITCH는 숫자든 문자든 받을 수 있고, 어떤 값일 때 무엇을 낼지 값을 직접 지정합니다.
언제 쓰나 과목 코드가 A·B·C처럼 문자면 CHOOSE는 숫자로 바꾸는 단계가 하나 더 필요합니다. SWITCH는 그 단계 없이 바로 씁니다.
참고: SWITCH와 IFS는 최근 추가된 함수로 아직 시험에 나온 적이 없습니다. 급하면 CHOOSE에 집중하세요.

RANK.EQ — 순위를 매긴다 (필기·실기 단골)

구조 =RANK.EQ(내 점수, 전체 범위, 옵션). 예: =RANK.EQ(D2,$D$2:$D$50,0)
순서 내 점수(단일 셀)가 먼저 오고, 그다음 비교할 전체 범위입니다. 내 점수 자리에 범위를 넣으면 안 됩니다.
옵션 0=내림차순(큰 값이 1등), 1=오름차순(작은 값이 1등). 생략하면 내림차순으로 처리됩니다. 점수·금액은 큰 값이 1등이라 내림차순, 달리기 기록(초)·등급은 작은 값이 1등이라 오름차순을 씁니다.
전체 범위는 F4 필수 절대 참조를 안 하면 채우기 핸들로 복사할 때 범위가 밀려 900만 원도 1등, 800만 원도 1등처럼 순위가 전부 망가집니다.
EQ와 AVG 동점 처리 방식 차이입니다. RANK.EQ는 같은 순위를 주고, RANK.AVG는 평균 순위를 줍니다. 컴활2급은 RANK.EQ만 씁니다.
시험 예시: 실기는 옵션 생략이 가능하지만, 필기 대비로 0을 명시하는 습관을 들이세요.

SMALL · LARGE · MEDIAN · MODE.SNGL — 통계값 꺼내기

SMALL / LARGE =SMALL(범위, k)는 k번째로 작은 값, =LARGE(범위, k)는 k번째로 큰 값. k=1이면 최저/최고입니다.
범위를 무조건 F4 하지 마세요 학생마다 자기 행의 국·영·수를 봐야 하면 절대 참조를 하면 안 됩니다. "범위=절대참조"로 외우지 말고 상황에 맞게 판단하세요.
총점은 빼기 "내 점수 중 제일 낮은 점수"를 구할 땐 총점을 범위에 넣지 않습니다. 총점은 응시한 개별 과목 점수가 아니니까요.
MEDIAN 데이터를 일렬로 세웠을 때 중앙값. 범위만 넣으면 끝입니다.
MODE.SNGL 가장 많이 등장한 값(최빈값). 범위만 넣으면 끝이고, 최빈값이 없으면 #N/A가 나옵니다 — 이건 오류가 아니라 정상 결과입니다.
시험 예시: SUMIF의 조건 자리에 LARGE·SMALL을 끼워 넣어 "상위 5위 이내 데이터의 합계" 같은 형태로 어렵게 냅니다.

IF — 조건식은 "질문"이다 (오늘의 주인공)

구조 =IF(조건식, 참일 때 값, 거짓일 때 값). 예: =IF(C2>=60,"합격","불합격")
조건식 = 질문 "너 60점 넘니?"를 엑셀이 알아듣게 코딩한 것이 C2>=60입니다. 주어(비교할 셀)가 반드시 앞에 옵니다.
답은 TRUE/FALSE 둘뿐 질문에 대한 답은 참·거짓밖에 없습니다. TRUE면 두 번째 인수를, FALSE면 세 번째 인수를 출력하고 즉시 종료합니다.
따옴표가 없다 SUMIF의 조건은 ">=80"처럼 따옴표가 필요합니다(비교 대상이 앞에 없고 조건 범위를 따로 주니까). 하지만 IF 조건식은 비교 대상이 바로 앞에 붙어 있어 따옴표를 쓰지 않습니다. 이 차이를 가장 헷갈려 합니다.
공백 출력 "불합격 대신 공백으로 나타내시오"가 자주 나옵니다. 따옴표 2개 ""를 쓰면 됩니다.
시험 예시: 컴활2급 필기 합격 기준(평균 60점)처럼 실제 기준을 그대로 조건식으로 옮겨 적는 연습을 하세요.

중첩 IF — 구간이 N개면 IF는 N-1개

먼저 구간을 센다 평균 90↑=A, 80↑=B, 70↑=C, 60↑=D, 그 외 F → 결과가 5개 구간. 그러면 질문은 4번만 하면 됩니다. 마지막 F는 자동으로 거짓 구간에 배정되니까요.
구조 90점 넘으면 A 출력 후 종료, 아니면 거짓 구간으로 가서 다시 "80점 넘니?" 질문. 거짓 구간 안에 다음 질문을 계속 쌓아 올리는 구조입니다.
재사용 패턴이 반복되므로 Ctrl+C → Ctrl+V로 붙여 넣고 숫자만 바꾸면 훨씬 빠릅니다.
괄호 몇 개를 닫아야 할지 세지 말고, 글자가 검정색이 될 때까지 닫으면 됩니다.
시험 예시: 근속 10년↑ 20%, 5년↑ 10%, 그 외 5% → 구간 3개이므로 IF는 2개. 문제에 "9년 이하" 같은 헷갈리는 멘트를 섞어 놓지만, 10년 이상 / 5년 이상 / 그 외만 보면 됩니다.

AND · OR — 질문을 동시에 여러 개 던질 때

왜 필요한가 IF 조건식에는 질문을 1개만 넣을 수 있습니다. "평균 80↑ 이고 각 과목 60↑"처럼 질문이 여러 개 동시에 필요하면 조건식 안에 AND·OR를 씁니다.
AND vs OR 조건을 모두 만족해야 하면 AND, 하나만 만족해도 되면 OR. 문제를 읽고 스스로 판별해야 합니다.
들어가는 자리 AND·OR은 IF의 조건식 자리에 들어갑니다. 예: =IF(AND(E2>=80,B2>=60,C2>=60),"합격","")
괄호 주의 ★ 가장 많이 틀리는 지점입니다. AND·OR의 괄호는 조건식 안에서 닫아야 합니다. 그 뒤에 참값·거짓값이 옵니다.
중첩 IF와 구분 구간을 나누는 문제면 중첩 IF, 질문을 동시에 던지는 문제면 AND·OR입니다. 시험은 보통 어떤 함수를 쓰라고 알려줍니다.
시험 예시: 재고 상태가 "부족"이거나 현재 재고가 최소 재고의 1.2배 이하면 발주 → 결과가 2개(필요·불필요)이므로 IF는 1개, 질문은 동시에 2개라 OR.

IFS · IFERROR — IF의 형제들

IFS 조건식·참값 쌍을 반복하며 거짓값 인수가 없습니다. 위에서부터 훑다가 TRUE를 만나면 출력하고 종료합니다. 마지막 "나머지"는 조건식에 TRUE(또는 1)를 적어 무조건 걸리게 만듭니다.
IFERROR =IFERROR(원래 수식, 오류일 때 값). 찾기·참조 함수에서 자주 나는 #N/A를 공백 ""이나 "코드 오류" 같은 문구로 바꿔 보기 좋게 만듭니다.
참고: IFS는 구버전(2016 등)에 없고 시험 출제 이력이 없습니다. IFERROR는 문제 풀면서 익혀도 충분합니다.

날짜·시간 함수 — 쪼개고 합치기

인수가 없는 함수 TODAY()(오늘 날짜), NOW()(현재 날짜+시간)는 인수가 아예 없습니다. 괄호만 열고 닫으면 끝.
단축키 Ctrl+;=오늘 날짜, Ctrl+Shift+;=현재 시간. "오늘 날짜를 넣는 방법으로 옳지 않은 것은?"이 필기에 나옵니다.
쪼개기 → 합치기 (날짜) YEAR·MONTH·DAY로 연·월·일을 각각 추출하고, DATE(년, 월, 일)로 다시 날짜 형식으로 합칩니다.
쪼개기 → 합치기 (시간) HOUR·MINUTE·SECOND로 시·분·초를 추출하고, TIME(시, 분, 초)으로 다시 시간 형식으로 합칩니다.
시험 예시: 동작은 단순한데 문제를 어렵게 냅니다. DATE·TIME이 어렵다는 평이 많으니 인수 순서(연·월·일 / 시·분·초)를 확실히 해두세요.

WEEKDAY — 요일을 숫자로 (★★ 아주 중요)

구조 =WEEKDAY(날짜, 옵션번호). 인수는 2개입니다. 요일을 숫자로 돌려줍니다.
옵션 2를 가장 많이 쓴다 옵션 2 = 월요일이 1, 화요일 2 … 일요일 7. 평일·주말 구분이 쉬워서입니다. (옵션 1은 일요일이 1) 옵션 번호는 시험에서 다 주니 외울 필요 없습니다.
활용 ① CHOOSE와 중첩 → 한글 요일 =CHOOSE(WEEKDAY(날짜,2),"월","화","수","목","금","토","일")
활용 ② IF와 결합 → 평일/주말 옵션 2에서 평일은 5 이하이므로 =IF(WEEKDAY(날짜,2)<=5,"평일","주말")
강사 코멘트: "IF 문제 하나를 40분이나 붙잡고 푼 문제가 바로 이 WEEKDAY였다." 날짜·시간 함수 중 가장 주의 깊게 봐야 할 함수입니다.

AND·OR의 괄호를 조건식 밖에서 닫아요

IF 안에서 AND·OR를 쓸 때 괄호를 엉뚱한 데서 닫아 수식이 통째로 꼬입니다. AND·OR는 조건식 안에서 끝나야 하고, 그 뒤에 참값·거짓값이 옵니다.

AND·OR는 조건식 자리에서 닫는다 — =IF(AND(조건1,조건2),"참","거짓")

자주 하는 실수

RANK.EQ 전체 범위에 F4를 안 걸어요

절대 참조를 안 하면 채우기 핸들로 복사할 때 범위가 밀려 900만 원도 1등, 800만 원도 1등이 됩니다(수업 중 실제로 났던 오류).

내 점수는 상대 참조 그대로, 전체 범위만 F4로 $D$2:$D$50 고정.

자주 하는 실수

IF 조건식에 따옴표를 넣어요

SUMIF의 조건은 ">=80"처럼 따옴표가 필요하지만, IF 조건식은 비교 대상이 바로 앞에 붙어 있어 따옴표가 없습니다.

IF 조건식 = 따옴표 없음 / SUMIF류 조건 = 따옴표 있음.

자주 하는 실수

중첩 IF에서 구간을 안 세고 바로 식을 써요

결과가 몇 개인지 세지 않고 무작정 쓰다가 IF 개수가 안 맞습니다. 결과 구간이 5개면 IF는 4개, 2개면 1개입니다.

① 결과 구간 수를 센다 → ② IF는 (구간-1)개 → ③ 그다음 식을 쓴다.

자주 하는 실수

"범위니까 무조건 절대 참조"라고 외워요

SMALL·LARGE에서 학생마다 자기 행의 국·영·수를 봐야 하면 절대 참조를 하면 안 됩니다. 반대로 CHOOSE의 기준표 셀은 단일 셀이라도 기준값이면 절대 참조해야 합니다.

"범위=절대참조"가 아니라 "기준이 되는 것=절대참조"로 판단하세요.

자주 하는 실수

이상·이하·초과·미만의 등호(=)를 빠뜨려요

"최소 재고 이하"인데 <만 쓰면 틀립니다. 강사 표현으로 "떼먹으면 땡"입니다.

이상 >= / 이하 <= / 초과 > / 미만 < — 등호 유무를 항상 확인.

자주 하는 실수

CHOOSE 첫 인수에 숫자가 아닌 걸 넣어요

CHOOSE의 첫 인수는 반드시 숫자이고 1번부터 시작합니다(0번 없음). 코드가 0부터 시작하면 +1로 맞춰야 합니다.

문자 코드(A·B·C)를 그대로 쓰고 싶다면 CHOOSE가 아니라 SWITCH.

자주 하는 실수

#N/A나 초록색 삼각형을 오류로 오해해요

MODE.SNGL은 최빈값이 없으면 정상적으로 #N/A가 납니다. 셀 왼쪽 위 초록색 삼각형은 "인접 셀 숫자를 빠뜨린 것 아니냐"는 알림일 뿐입니다.

의도한 것이면 "오류 무시"로 표식 제거. 진짜 #N/A를 감추려면 IFERROR.

자주 하는 실수

긴 수식에서 지금 어느 인수인지 놓쳐요

IF가 길어지면 콤마가 엉뚱한 데 들어가 꼬입니다. 게다가 실수를 찾는 게 만드는 것보다 훨씬 힘듭니다.

스크린팁으로 현재 위치를 확인하거나, 아예 대화상자(fx)를 쓰면 인수가 칸별로 나뉘어 안 꼬입니다.

자주 하는 실수

표시 형식이 텍스트라 결과가 안 나와요

셀이 텍스트 형식이면 함수를 제대로 써도 결과가 글자로 그대로 보입니다.

표시 형식을 일반으로 바꾼 뒤 F2 → Enter로 다시 확정하세요.
