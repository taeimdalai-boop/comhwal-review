📊 차트=혼합(콤보)+보조축 ✂️ 텍스트 나누기=쉼표 구분 ⚙️ 매크로=절대 참조로 기록 ⏱ 접근시간=밀마나피펨아 💾 SSD=반도체·충격 강함 🖥 OS=감작대·언서문

⚡ 꼭 맞히고 가요 — 진법 변환 실전 문제 (수강생 제보)

권나경 학생 지인분이 최근 시험에서 여러 번 만난 유형이에요. 진법 변환은 요즘 계속 출제되고 있으니, 이 문제 하나는 반드시 잡고 갑니다!

다음 중 진수 변환에 대한 설명으로 옳은 것을 고르시오.

① 10진수 15를 16진수로 변환하면 E이다  ✕ (E=14)
② 10진수 15를 16진수로 변환하면 F이다  ✓ 정답
③ 16진수 A를 10진수로 변환하면 9이다  ✕
④ 16진수 A를 10진수로 변환하면 11이다  ✕ (A=10)
왜 ②인가? 16진수는 0~9 다음 10=A · 11=B · 12=C · 13=D · 14=E · 15=F. 그래서 10진수 15 = F(①의 E는 14라 오답). 거꾸로 16진수 A는 10진수로 10이라 ③(9)·④(11) 모두 오답. A~F = 10~15만 확실히 외우면 끝!

📊 차트 — 구성 요소와 편집 (마지막 단원)

제목을 셀과 연결 제목 선택(실선)→수식 입력줄에 =A1 입력→Enter. 데이터 레이블은 특정 계열만 두 번 클릭(선택점 하나만)→바깥쪽 끝.
시험: 차트 5문제(각 2점). 요소 명칭·추가/제거·제목 연결·데이터 레이블이 단골.

🔀 혼합(콤보) 차트 · 범위 조절 · 축 서식

혼합 차트 수치 차이가 큰 계열(불량률)은 잘 안 보여요. 계열 하나 선택→차트 종류 변경→맨 아래 혼합→해당 계열을 표식 있는 꺾은선형 + 보조축 체크. 시험엔 "콤보"라고 나옵니다.
범위 조절 차트 선택 시 원본에 파란 선(연속 범위만 보임). 모서리를 드래그해 조절. 계열 제거=계열 선택→Delete. 중간 계열 제거=행렬 전환→Delete→다시 행렬 전환.
축 서식 축을 더블 클릭→오른쪽 작업창 축 옵션에서 최소/최대값·기본 단위(눈금 간격) 조절. 그림 영역 채우기=단색·그라데이션(미리 설정·종류·방향)·질감·패턴.
시험: "행렬 전환하면? → 가로 항목축과 범례 위치가 바뀐다". 보조축·기본 단위 조절이 자주 나옵니다.

✂️ 텍스트 나누기 · ⚙️ 매크로(복습)

텍스트 나누기 CSV처럼 쉼표로 한 셀에 붙어 있는 값을 열로 분리. 범위 선택→데이터 탭→텍스트 나누기→"구분 기호로 분리"→쉼표 체크(탭 해제)→특정 필드는 열 가져오지 않음→마침.
매크로 순서 ⓪셀 포인터 바깥 이동 → ①기록 → ②동작 → ③중지 → ④도형/단추에 지정 → ⑤검증.
절대 참조로 기록 매크로 기록 시 "상대 참조로 기록"은 반드시 꺼짐이어야 해요. 켜지면 실행 위치에 따라 엉뚱한 셀에 결과가 나옵니다. 단추는 그리면 바로 매크로 상자, 도형은 우클릭→매크로 지정.
시험: 매크로 5점×2. 셀 포인터 위치와 무관하게 실행돼야 정답(절대 참조 기록).

💾 기억장치 · 보조기억장치 (필기)

접근 시간 값이 작을수록 빠름. ms(10⁻³)>µs>ns>ps>fs>as(10⁻¹⁸). 3승씩 작아지고 아토세컨드(as)가 가장 빠름. "밀마나피펨아"로 암기.
용량 단위 B<KB<MB<GB<TB<PB<XB (2¹⁰=1024배씩).
SSD vs HDD HDD=원판·트랙·섹터·헤드 회전·충격 약함(고정 사용). SSD=반도체·빠름·무소음·저전력·충격 강함·가벼움(단점=비쌈). 연결 방식 SATA=직렬(시리얼)·핫플러그/핫스왑.
기타 파티션(물리적/논리적, OS 여러 개 설치), RAID(여러 디스크를 논리적으로 하나처럼), 가상 메모리=보조기억 일부를 주기억처럼, 캐시=SRAM, ROM(비휘발성·POST)·RAM(휘발성), 플래시=EEPROM.
시험: "가장 빠른 접근 시간 = as", "SSD 특징(충격 강함·무소음)", "SATA=직렬" 단골.

🔌 포트·장치·전원 (필기)

USB 직렬(한 bit씩)·최대 127개·핫플러그·PnP. 버전 색상: 검정=구형 / 파랑=신형(속도↑). 직렬/병렬 PS2를 하나로 통합.
포트/전송 HDMI=영상+음향 한 케이블 통합. 채널=입출력을 CPU 대신 위임. DMA=CPU 개입 없이 메모리 직접 전송. 버스=데이터·주소·제어. 인터럽트=이벤트 알림(IRQ). 데드락=서로 자원 대기(무한 대기).
입출력 장치 입력=키보드·스캐너·OMR·OCR(손글씨 인식)·MICR(수표)·키오스크 / 출력=모니터·프린터(도트·잉크젯·레이저=토너).
전원 장치 UPS=무정전 전원 공급(정전 대비), AVR=자동 전압 조절, 항온항습기(온·습도), 서지 보호기(전압 급변 차단). POST 비프음: 길게1 짧게3=그래픽 카드.
시험: "정전 대비 전원 공급=UPS", "입출력 위임=채널", "CPU 없이 직접 전송=DMA".

🖥 소프트웨어 · 운영체제 (필기)

분류 시스템 SW=운영체제(윈도우·리눅스·유닉스·맥OS·안드로이드·iOS) / 응용 SW=워드·엑셀·파워포인트. 펌웨어=롬에 새긴 SW(하드웨어 제어, 예 바이오스).
운영체제 구성 제어 프로그램 = 감작대(감시·작업 관리·데이터 관리) + 처리 프로그램 = 언서문(언어 번역·서비스·문제 처리). "제어(ㅈ)가 먼저, 처리(ㅊ)가 뒤".
임베디드 OS 밥솥·냉장고 등 가전에 들어가는 소형 운영체제(윈도우 CE·임베디드 리눅스·안드로이드).
시험: "처리 프로그램이 아닌 것?"에 데이터 관리(제어)를 끼워 넣어 함정. 감작대/언서문 구분이 핵심.

접근 시간이 클수록 빠르다고 생각해요

접근 시간은 값이 작을수록 빠릅니다. 가장 빠른 건 as(아토세컨드).

ms>µs>ns>ps>fs>as 중 as가 가장 빠름(10⁻¹⁸).

필기 · 자주 틀림

USB·SATA를 병렬 방식이라고 생각해요

USB·SATA는 직렬(시리얼)이에요. 한 번에 한 bit씩 보냅니다.

직렬=한 bit씩 / 병렬=여러 bit 동시(예 옛 LPT). USB=직렬·최대 127개.

필기 · 자주 틀림

SSD가 충격에 약하다고 생각해요

원판이 도는 HDD가 충격에 약하고, 반도체 SSD가 충격에 강하고 무소음이에요.

SSD=충격 강함·무소음·빠름(단점=비쌈) / HDD=충격 약함.

필기 · 자주 틀림

데이터 관리를 "처리 프로그램"에 넣어요

데이터 관리는 제어 프로그램(감작대)입니다. 처리 프로그램은 언서문.

제어=감시·작업 관리·데이터 관리 / 처리=언어 번역·서비스·문제 처리.

필기 · 자주 틀림

정전 대비 장치를 AVR이라고 골라요

정전 시 전원을 공급하는 건 UPS. AVR은 전압을 일정하게 조절하는 것.

정전 대비 전원 공급=UPS(무정전 전원 공급 장치).
