오늘의 핵심은 "시간 계산은 연산 먼저 HOUR, 조건부 서식으로 행 전체를 칠하려면 열 절대참조($A5)"입니다.

2026-07-20 · 10차시 · 함수 마무리 + 필기 첫날입니다. 주차 요금에서 시간은 빼고 나서 HOUR/MINUTE로 감싸야 맞고, 날짜는 반대로 YEAR로 따로 뽑아 뺍니다. 조건부 서식은 IF의 조건식과 같고, 행 전체 강조는 열만 고정합니다. 필기 컴퓨터 일반(GUI/CLI/NUI, PnP, OLE, 파일 시스템)도 시작했습니다.

⏱ 시간=연산 후 HOUR 📅 날짜=YEAR 따로 🎨 조건부 서식(수식) 🔒 행 전체=열 절대($A5) 🔤 문자 vs 숫자 맞추기 🖥 필기: Windows 10

📌오늘 배운 핵심 요약

⏱ 시간 계산 = 연산 먼저

주차·이용 요금은 셀을 먼저 빼고 그 결과를 HOUR·MINUTE로 감쌉니다. =HOUR(출차-입차). 시·분을 각각 따로 빼면 50분도 1시간으로 잘못 계산돼요.

시간=연산 후 감싸기 / 결과 안 보이면 표시형식 일반

📅 날짜 계산 = 따로 뽑기

나이 등은 반대로 YEAR/MONTH/DAY로 각각 추출해서 뺍니다. =YEAR(오늘)-YEAR(생일). 시간과 정반대라 헷갈리기 쉬워요.

시간=연산 후 / 날짜=따로 추출

🚗 주차 요금 완성식

분으로 환산 후 10분당 200원: =(HOUR(C2-B2)*60+MINUTE(C2-B2))/10*200. 시간×60+분 → ÷10 → ×200.

전체를 괄호로 감싸고 ÷10 ×200

🎨 조건부 서식 = IF 조건식

새 규칙 → "수식을 사용하여 서식 지정". 여기 넣는 건 IF의 조건식(질문)과 똑같아요. 참(TRUE)인 셀에만 서식이 적용됩니다.

조건부 = 이프(IF) · 참일 때만 색칠

🔒 행 전체 강조 = 열 절대

조건인 행 전체를 칠하려면 조건 열만 고정합니다(열 절대참조). =LEFT($A5,4)="2024" — A만 $로 묶고 행 번호는 안 묶어요.

행 전체 서식 → 열 절대($A5) (공식 암기)

🔤 문자 vs 숫자 맞추기

LEFT·MID·RIGHT 결과는 문자라 숫자와 그냥 비교하면 안 맞아요. ="2024"(문자로) 또는 LEFT(...)*1(숫자로) — 둘 다 알아두기.

문자=따옴표 / 숫자=곱하기 1

🧩 ROW·COLUMN 함수

ROW=행 번호, ROWS(범위)=행 개수, COLUMN=열 번호, COLUMNS(범위)=열 개수. 조건부 서식 줄무늬·체스판(MOD)에 씁니다.

번호(ROW/COLUMN) vs 개수(ROWS/COLUMNS)

🖥 필기: Windows 10 첫날

인터페이스 GUI/CLI/NUI, 선점형 멀티태스킹, PnP(연결하면 동작), OLE(엑셀→PPT 삽입), 파일 시스템(FAT·NTFS·exFAT).

필기 컴퓨터 일반 시작 — 용어 암기

💡개념 다시 보기

⏱ 시간 계산 — 연산 먼저, HOUR/MINUTE 나중 (오늘 최고 강조)

규칙 주차·이용 요금처럼 "몇 시간·몇 분 썼나"를 구할 땐 셀을 먼저 빼고 그 결과를 감쌉니다. =HOUR(출차-입차), =MINUTE(출차-입차).
왜 시·분을 각각 따로 추출해서 빼면(11시-10시=1시간) 실제 10:30~11:20(=50분)인데 1시간으로 계산돼 틀립니다. 시간 데이터가 0~1 소수 형식이라 생기는 함정.
완성식 10분당 200원: =(HOUR(C2-B2)*60 + MINUTE(C2-B2)) / 10 * 200. 분으로 환산 → ÷10 → ×200.
표시 형식 결과가 시간처럼 이상하게 보이면 표시 형식을 일반으로 바꾸고 F2 → Enter.
시험 예시: 주차 요금·이용 요금 문제는 결과가 나와도 방법이 틀리면 값이 어긋납니다. "시간=연산 후 감싸기"를 꼭 외우세요.

📅 날짜 계산 — 따로 추출해서 빼기 (시간과 반대)

규칙 나이·근속처럼 날짜 차이는 YEAR/MONTH/DAY로 각각 뽑아 계산합니다. =YEAR(오늘)-YEAR(생일).
대비 시간은 "연산 후 감싸기", 날짜는 "따로 추출해 빼기" — 정반대입니다. 이 둘을 세트로 외우세요.
시험 예시: 날짜를 시간처럼 먼저 빼고 YEAR로 감싸면 틀립니다.

🔎 WEEKDAY 중첩 — 찾기 함수의 검색값 자리에

구조 VLOOKUP/HLOOKUP의 검색값 자리에 WEEKDAY를 끼웁니다. =VLOOKUP(WEEKDAY(날짜,2), 기준표, 2, FALSE).
일치 옵션 요일 번호는 코드처럼 딱 떨어지므로 마지막 인수는 FALSE(정확히 일치). 기준표는 채우기 대비 F4 절대참조.
시험 예시: 찾기 함수는 검색값 자리에 다른 함수(WEEKDAY·LEFT·MID 등)를 중첩하는 유형이 잘 나옵니다.

🎨 조건부 서식 — "수식을 사용하여 서식 지정" = IF의 조건식

1단계 영역 선택(실기) 새 규칙을 만들기 전에 머리글(필드명)은 빼고 레코드 범위만 선택합니다. 실무는 상관없지만 시험은 지정 영역이 정해져 있어요.
2단계 규칙 조건부 서식 → 새 규칙 → "수식을 사용하여 서식을 지정할 셀 결정"을 고릅니다(실기 단골).
3단계 조건식 여기 넣는 건 IF의 조건식(질문)과 같습니다. 예: 2024학번이면 =LEFT($A5,4)="2024". 참(TRUE)인 셀에만 서식이 적용됩니다.
서식 글꼴/채우기에서 색을 지정. "글꼴색"인지 "채우기색"인지 문제를 잘 보세요. 스크린팁이 안 뜨면 무늬(3번 탭)에서 색 위치를 참고합니다.
시험 예시: 5·6월 기수는 이 문제가 전원 0점이 나올 만큼 어렵습니다(강사 경고). 꼭 연습하세요.

🔒 행 전체 강조 = 열 절대참조 / 문자 vs 숫자 맞추기

열 절대 조건에 맞는 행 전체를 칠하려면 조건이 되는 열만 고정합니다. $A5 — A(열)만 $로 묶고 행 번호는 안 묶어요(아래로 내려가며 검사해야 하니까). "행 전체 서식 → 열 절대"는 공식으로 외웁니다.
왜 전부 고정하면 안 되나 $A$5로 다 묶으면 A5 한 칸만 계속 보게 돼 다른 행을 검사하지 못합니다.
문자 vs 숫자 LEFT/MID/RIGHT는 텍스트 추출 함수라 결과가 문자. 숫자 2024와 비교하면 안 맞습니다. 문자로 맞추기 ="2024" 또는 숫자로 맞추기 LEFT(...)*1(×1로 숫자화). 둘 다 알아야 합니다.
시험 예시: LEFT로 앞 4자리를 뽑아 "2024"와 비교하는 조건부 서식. 열 절대 + 문자 맞추기 둘 다 걸립니다.

🧩 ROW / ROWS / COLUMN / COLUMNS + 조건부 서식 도구

ROW / COLUMN 행 번호 / 열 번호를 반환. 인수를 생략하면 현재 셀 기준. (셀 하나만 참조)
ROWS / COLUMNS 범위를 넣으면 그 범위의 행 개수 / 열 개수를 반환. 레코드 개수를 셀 때 COUNT 대신 쓸 수 있어요.
줄무늬·체스판 ROW와 MOD를 조합하면 짝수/홀수 행만 색칠(줄무늬), 행+열 조합으로 체스판을 만들 수 있습니다(응용).
대화상자 함수명 안 뜰 때 조건부 서식 수식 칸은 자동완성이 안 됩니다. 셀에 미리 수식을 적어 검증 → Ctrl+X로 잘라 → 규칙 수식 칸에 붙여넣기.
상위/하위·아이콘 홈 → 조건부 서식 → 상위/하위 규칙(상위 10%만 강조 등), 데이터 막대·색조·아이콘 집합. 정렬에서도 조건부 서식 아이콘 기준으로 정렬됩니다.
시험 예시: ROW·MOD로 줄무늬 만들기, 상위 N%/N개 강조가 필기·실기에 나옵니다.

🖥 필기 컴퓨터 일반 — Windows 10 (첫날)

사용자 인터페이스 GUI(그래픽·아이콘·마우스), CLI(명령어 입력 — 요즘 바이브 코딩으로 다시 많이 씀), NUI(터치·음성 등 자연스러운 방식).
선점형 멀티태스킹 운영체제가 각 작업의 CPU 사용 시간을 강제로 배분·통제합니다. 한 프로그램이 CPU를 독점하지 못하게 OS가 뺏어 옵니다.
플러그 앤 플레이(PnP) 주변장치를 연결하면 자동으로 인식·동작하도록 설정. "연결한다 → 동작한다".
OLE(개체 연결 및 삽입) 엑셀에서 만든 문서·차트를 파워포인트·워드에 넣는 것처럼 다른 앱의 개체를 연결·삽입.
파일 시스템 옛 FAT16/FAT32, NTFS(윈도우 전용), 요즘 쓰는 exFAT(FAT의 용량 한계 개선, USB·SD 등 이동식에 적합, 윈도우·맥 호환).
단축키 Win+D(바탕화면), Win+E(탐색기), Win+L(잠금), Alt+Tab(창 전환), Alt+F4(닫기), Win+Tab(작업 보기), F2(이름 바꾸기), F5(새로 고침).
시험 예시: 용어 정의(선점형·PnP·OLE)와 파일 시스템 특징을 묻는 문제가 필기 단골입니다.

🚨자주 하는 실수

⭐ 가장 많이 틀린 부분

시간 요금에서 시·분을 각각 따로 빼서 계산해요

11시-10시=1시간처럼 나와, 실제 10:30~11:20(50분)인데 1시간으로 잘못 계산됩니다. 결과가 나오니 맞은 줄 알고 넘어가 버려요.

반드시 =HOUR(출차-입차)처럼 먼저 빼고 HOUR/MINUTE로 감싸기.

자주 하는 실수

조건부 서식에서 행 전체를 열 절대($A5)로 안 묶어요

전부 고정($A$5)하면 A5 한 칸만 보고, 안 묶으면 열이 밀립니다. 그러면 색칠이 엉뚱하게 돼요.

행 전체 강조는 조건 열만 고정 — $A5 (A만 $).

자주 하는 실수

LEFT 결과(문자)를 숫자와 그냥 비교해요

LEFT·MID·RIGHT는 텍스트 추출 함수라 결과가 문자입니다. 숫자 2024와 비교하면 안 맞아요.

="2024"(문자로 맞추기) 또는 LEFT(...)*1(숫자로 맞추기).

자주 하는 실수

날짜를 시간처럼, 시간을 날짜처럼 계산해요

날짜를 먼저 빼고 YEAR로 감싸거나, 시간을 따로 추출해서 빼면 틀립니다.

시간=연산 후 감싸기 / 날짜=따로 추출해 빼기. 정반대로 외우기.

자주 하는 실수

조건부 서식 범위에 머리글(필드명)을 넣어요

실무는 상관없지만 실기 시험은 지정 영역이 정해져 있어 감점됩니다.

머리글 빼고 레코드 범위만 선택.

자주 하는 실수

IF 조건식 안에 여러 셀을 배열로 넣어요

"셋 중 하나라도 80 이상"을 IF 조건에 배열로 넣으면 IF 단독으로는 안 됩니다.

여러 조건 동시엔 OR(하나만)·AND(모두) 함수를 조건식 안에.

자주 하는 실수

주차 요금 결과가 시간 형식으로 보여요

계산은 맞는데 표시가 시간(0.xx)으로 나옵니다.

표시 형식을 일반으로 바꾸고 F2 → Enter로 재확정.

자주 하는 실수

ROW와 ROWS, COLUMN과 COLUMNS를 헷갈려요

ROW/COLUMN은 번호(셀 하나), ROWS/COLUMNS는 개수(범위)입니다.

"번호 vs 개수" — 복수형(S)이 붙으면 개수.

자주 하는 실수

필기 용어를 대충 암기해요

선점형↔비선점형, PnP·OLE·파일 시스템 특징을 헷갈립니다.

PnP=연결하면 동작 / OLE=문서·차트 삽입 / exFAT=이동식·대용량으로 딱 붙여 외우기.

자주 하는 실수

대화상자에서 함수명이 안 떠서 못 써요

조건부 서식 수식 칸은 자동완성이 안 됩니다.

셀에 미리 수식을 적어 검증 → Ctrl+X로 잘라 규칙 칸에 붙여넣기.
